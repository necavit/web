  • Response

get /users/{userId}

get

Retrieve a user's public information.

get /users/{userId}

Retrieve a user's public information.

  • Request
  • Response

URI Parameters

  • userId: string

HTTP status code 200

The user's information is available.

Body

Type: application/json

Example: 
{
  "userId" : "123456789abcdefg",
  "displayName" : "John Doe",
  "type" : "human",
  "statistics" : {
    "killed" : 10,
    "hidden" : 0
    "level" : 10
  }
}

HTTP status code 400

Bad request, if the given userId does not match with any existing user.

post /users/{userId}/attacks

post

Creates a new Attack against another user and returns the result of the attack.

post /users/{userId}/attacks

Creates a new Attack against another user and returns the result of the attack.

  • Request
  • Response

URI Parameters

  • userId: string

Body

Type: application/json

Example: 
{
  "against" : {
    "userId" : "123456abcdef"
  }
}

HTTP status code 201

The attack was successfully made. The result is available in the response body.

Body

Type: application/json

Example: 
{
  "resultCode" : "success",
  "updatedUser" : {
    "statistics" : {
      "killed" : 11,
      "hidden" : 0
      "level" : 11
    },
    "type" : "human"
  }
}

{
  "resultCode" : "failure",
  "updatedUser" : {
    "statistics" : {
      "killed" : 10,
      "hidden" : 0
      "level" : 9
    },
    "type" : "infected"
  }
}

HTTP status code 400

Bad request, if the requested attack cannot be performed or either the attacker or the defender don't exist as users.

Body

Type: application/json

Example: 
{
  "errorType" : "no such user exists",
  "reason" : "no user exists with the given userId"
}

{
  "errorType" : "bad attack request",
  "reason" : "attack recipient out of range"
}

{
  "errorType" : "bad attack request",
  "reason" : "attack recipient is of the same kind as the attacker"
}
